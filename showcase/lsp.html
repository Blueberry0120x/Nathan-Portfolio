The arc

From a folder nobody could navigate to a live catalog

A CAD team accumulates LISP the way a garage accumulates tools — powerful, but scattered, undocumented, and impossible to search. The best routines get buried; the same command gets re-written because no one knew it existed. The fix was not more code — it was making the library legible.

Zero · the scattered state

Routines with no map

Dozens of .lsp files across an evolving master library. No search, no categories, uneven headers, no way to see what exists or how commands relate. Discovery by tribal knowledge.

Hero · the catalog

One JSON, two synchronized viewers

A pipeline parses every (defun c:…) block into a structured catalog, then generates a searchable list and a 3D command map from the same data — so the two views can never disagree.

The build, honestly

A four-stage harvester, not a hand-made list

The catalog is generated, never hand-edited — so it stays true to the source every time it runs. Four stages, each guarding against a different way a hand-maintained index goes stale.

Sync
Mirror the source, read-only. One-way copy of the LISP master library into a working mirror — the source is never touched or risked.
Inspect
Drift report. Diff source against the mirror so the catalog can never silently fall behind the real library.
Stage
Filter to canonical. Keep the library files that belong in the catalog and drop scratch and legacy noise.
Build
Parse → JSON → viewers. Read every (defun c:…) into CommandLibrary.json, then render both viewers and the standards scorecard from it.
