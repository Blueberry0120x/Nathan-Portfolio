The arc

From a manual chore to a synchronized engine

Keynotes look trivial — a number on the drawing, a matching row in a table. In a real sheet set they are neither. Renumber one callout and every downstream table drifts; add a note and someone re-types the schedule. On a large civil package that is hours of senior-designer time and a standing source of QA errors.

Zero · the manual state

Callout by hand, table by hand

Designers place keynote tags one at a time, then hand-build and hand-maintain the keynote table. Every renumber, insertion, or note edit means re-checking the whole schedule. Slow, and easy to get wrong at QA.

Hero · the engine

One source of truth, two-way sync

A Dynamo engine reads the drawing and writes the table — and reads the table back to the drawing. Renumber once; tags and schedule stay consistent automatically. The designer curates content instead of copying it.

The build, honestly

Five versions of learning in public

The interesting part for an AU audience is not the finished graph — it is the road to it. Each version solved a failure the last one exposed. That evolution is the teachable content.

v2.0
Callout automation. Place and number keynote tags from a controlled list — kill the one-at-a-time placement.
v3.0
Table generation. Build the keynote table from the tags in the drawing. One direction: drawing → table.
v4.0
Bi-directional sync. Read the table back to the drawing so an edit in either place reconciles — the hard problem, and the heart of the class.
v5.0
Generalized engine. Refactored into a reusable Dynamo tool (DynamicTag / TableNote) so the pattern applies beyond one project's standards.

Keynote Suite: Bi-Directional Keynote & Table Automation in Civil 3D with Dynamo
Session format · track · level
60-minute Instructional Demo  ·  Civil Infrastructure (AEC)  ·  Intermediate
Abstract
Keynotes are one of the last hand-maintained links in Civil 3D sheet production: a tag on the drawing, a matching row in a table, and a manual re-check every time either changes. This class walks through a production Dynamo tool — the Keynote Suite — that automates keynote callouts and their tables and, critically, keeps them synchronized in both directions. Edit a note in the drawing and the schedule updates; adjust the schedule and the drawing follows. We trace the tool across five real versions, from first callout automation to a generalized, reusable engine, and show the measured 60–80% reduction in keynote-and-revision time on live sheet sets. You will leave with the design pattern, the failure modes we hit, and a clear path to build the same two-way link into your own standards — no prior Dynamo mastery required.
Learning objectives — attendees will be able to
  1. Diagnose where manual keynote and table workflows cost time and introduce QA errors in a Civil 3D sheet set.
  2. Explain the bi-directional sync pattern that keeps drawing tags and schedule tables consistent from a single source of truth.
  3. Build the core of a keynote-automation Dynamo graph, including reading from and writing back to the drawing.
  4. Adapt the engine to their own office standards and estimate the time savings on their real projects.
